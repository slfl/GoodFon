# GoodFon

**Автосмена обоев рабочего стола для Windows**

Небольшое нативное приложение, которое живёт в трее и само меняет обои,
подбирая случайные картинки с [goodfon.com](https://www.goodfon.com) / [goodfon.ru](https://www.goodfon.ru).

Написано на чистом **C** с использованием только **WinAPI** — без сторонних
runtime и зависимостей. Один exe весом около четверти мегабайта: живёт в трее,
почти не ест ресурсов, все настройки хранит в реестре. Версия файла и копирайт
зашиты в exe и видны в свойствах файла Windows.

## ✨ Возможности

**Обои**
- 🕒 Автосмена по таймеру — интервал от 5 минут до 24 часов.
- 🎨 35 встроенных тем — природа, город, космос, животные, авиация, абстракции, игры, ai-art и другие.
- 🖼 Выбор разрешения по тирам — HD / Full HD / 2K / 4K / 8K / «Оригинал».
- 🔙 История с кнопкой «Вернуть прошлые обои» — браузерная логика «назад», без повторов.
- 🖥 Несколько мониторов — на каждый экран своя картинка (режим «заполнение»).

**Избранное**
- ⭐ Понравившиеся обои → в личную папку и в избранное на сайте.
- 🔄 Каждая N-я смена берётся из избранного (5 / 10 / 15 / 20).
- 🧹 Тихая синхронизация при старте — удаляет локально то, что убрано с сайта (осторожно, с защитой от ошибок).

**Интерфейс**
- 🌗 Светлая и тёмная темы в стиле Windows 10/11 (тёмный заголовок через DWM).
- 🧭 Окно настроек с навигацией слева, owner-draw кнопки и списки.
- 🔔 Разные иконки уведомлений по типу события.
- 🖱 Меню трея с карточкой текущей картинки (клик → страница на сайте), иконками и скруглением.

**Система**
- 🔐 Пароль зашифрован через Windows DPAPI (не открытый текст, не украсть с другой машины).
- 🗄 Все настройки — в реестре `HKCU\Software\GoodFon`.
- 🚀 Автозапуск с Windows (ветка текущего пользователя, без прав администратора).
- ⬆️ Автообновление с GitHub по сравнению хэшей.
- 🌐 Полная локализация RU / EN (интерфейс и логи).
- 🤫 Логи только по флагу `-debug`.

## 🚀 Установка и запуск

1. Скопируйте `GoodFon-x64.exe` (или `GoodFon-x86.exe` для 32-битной системы) в любую папку.
2. Запустите двойным кликом — появится иконка в трее.
3. Правый клик → **«Настройки» → «Аккаунт»** — введите логин и пароль goodfon
   (нужны для разделов с авторизацией и для повышенного дневного лимита загрузок).
4. Там же выберите тему, разрешение и интервал.

При первом запуске, если в реестре нет настроек, создаются значения по умолчанию
(тема `nature`, Full HD, интервал 10 минут).

## ⚙️ Окно настроек

Одно окно с навигацией слева; правая часть сменяется. Изменения применяются сразу.

| Раздел | Содержимое |
|---|---|
| **Картинки** | Тема, интервал (5 мин…24 ч), разрешение, «из избранного», сколько временных картинок хранить, папка картинок |
| **Аккаунт** | Логин, пароль, «Войти» / «Выйти», ссылка «Регистрация». После входа — блок профиля с сайта: аватар, имя, рейтинг, счётчики (обои / скачивания / комментарии) |
| **Дополнительно** | Уведомления, автозапуск, домен (авто / .ru / .com), язык (RU / EN), оформление (светлая / тёмная) |
| **Обновления** | Текущая версия, автопроверка (выкл / час / день / неделя), проверка при запуске, автоустановка, уведомления, кнопка «Проверить обновления» |
| **О приложении** | Описание, версия, копирайт |

**Статистика профиля.** После входа на странице «Аккаунт» подгружается блок с
сайта (аватар 32×32, имя, рейтинг, счётчики). Данные кэшируются в реестре и
показываются мгновенно, а в фоне перечитываются и обновляются. Аватар в формате
WebP декодируется через WIC; если декод не удался — показывается иконка приложения.

## 🖼 Разрешение: как работают тиры

Приложение берёт разрешение, наиболее близкое к выбранному (не ниже, если есть),
а панорамные / ультраширокие варианты отсекает по соотношению сторон.

| Выбрано | Диапазон |
|---|---|
| HD | `1280×720` и близкие |
| Full HD | `1920×1080` |
| 2K | `2560×1440` |
| 4K | `3840×2160` |
| 8K | `7680×4320` |
| Оригинал | максимальное доступное |

Если в теме нет подходящей картинки — она пропускается, поиск продолжается
(до 40 изображений за цикл).

## 🗄 Хранение настроек — реестр `HKCU\Software\GoodFon`

| Значение | Тип | Назначение |
|---|---|---|
| `login` | REG_SZ | Логин goodfon |
| `password_enc` | REG_BINARY | Пароль, зашифрованный DPAPI |
| `session_com`, `session_ru` | REG_SZ | Кэш cookie-сессии по доменам |
| `resolution` | REG_SZ | Выбранное разрешение |
| `theme` | REG_SZ | Активная тема (slug) |
| `save_dir` | REG_SZ | Папка сохранения |
| `domain` | REG_SZ | Предпочтение домена: `com` / `ru` / `auto` |
| `max_files` | REG_DWORD | Сколько скачанных файлов хранить |
| `favorite_every_n` | REG_DWORD | Каждая N-я картинка — из избранного |
| `max_attempts` | REG_DWORD | Базовое число сетевых попыток |
| `notify` | REG_DWORD | Всплывающие уведомления (0/1) |
| `interval_min` | REG_DWORD | Интервал автосмены, минут |
| `update_interval_min` | REG_DWORD | Автопроверка обновлений, минут |
| `auto_update` | REG_DWORD | Ставить обновления автоматически (0/1) |
| `update_notify` | REG_DWORD | Уведомлять о новых версиях (0/1) |
| `check_on_startup` | REG_DWORD | Проверять обновления при запуске (0/1) |
| `profile_stats` | REG_SZ | Кэш статистики профиля |
| `counter` | REG_DWORD | Внутренний счётчик смен |
| `Language` | REG_SZ | Язык: `russian` / `english` |
| `AppTheme` | REG_SZ | Тема окна: `light` / `dark` |

**О безопасности пароля.** Пароль хранится зашифрованным через Windows DPAPI
(`CryptProtectData` / `CryptUnprotectData`), а не в открытом виде и не как хэш:
сайту при входе нужен настоящий пароль, а DPAPI держит его в защищённом виде
(привязка к учётной записи Windows) и расшифровывает только под вашей учёткой.

## 🔄 Синхронизация избранного

При каждом запуске приложение один раз сверяет локальную папку `Favorite\` с
избранным аккаунта на сайте и **удаляет локально то, чего больше нет на сайте**
(одностороннее удаление). Ничего не скачивает; сопоставление по имени файла.

Сделано максимально осторожно: выполняется только при успешном входе, список
читается со всех страниц (страницы, отданные как `502`, дочитываются повторно), и
**при любом сомнении синхронизация отменяется целиком** — лучше пропустить, чем
удалить лишнее. Процесс тихий, детали видны при `-debug`.

## ⬆️ Обновление с GitHub

Пункт «Проверить обновления» сравнивает **хэш** (FNV-1a 64)

Если хэши различаются — проверяется сигнатура `MZ` и размер, затем установка с
заменой работающего файла: exe переименовывается в `<exe>.old`, на его место
кладётся новый, запускается новая версия. На следующем старте `<exe>.old`
удаляется. Отдельный файл версий не нужен.

## 🔧 Сборка из исходников

Исходники — в папке `src/`. Код собирается и как **C**, и как **C++**.

### MinGW-w64 (как собраны эти бинарники)

```sh
# общие флаги и библиотеки
FLAGS="-municode -mwindows -Os -s -ffunction-sections -fdata-sections \
  -fno-asynchronous-unwind-tables -fno-unwind-tables -fno-ident \
  -fmerge-all-constants -Wl,--gc-sections -Wall"
LIBS="-lwinhttp -lshlwapi -lole32 -lshell32 -luser32 -ladvapi32 -lgdi32 \
  -lcrypt32 -luxtheme -ldwmapi -lwindowscodecs -lversion -lwinmm"

# x64
x86_64-w64-mingw32-windres resource.rc -O coff -o res.o
x86_64-w64-mingw32-gcc $FLAGS -o GoodFon-x64.exe main.c res.o $LIBS

# x86
i686-w64-mingw32-windres resource.rc -O coff -o res.o
i686-w64-mingw32-gcc $FLAGS -o GoodFon-x86.exe main.c res.o $LIBS
```

### Visual Studio + CMake

```bat
cd src
cmake -B build -A x64
cmake --build build --config Release
```

Для 32-бит: `cmake -B build32 -A Win32`. Включите `/utf-8` — в исходнике есть
русские строковые литералы. `goodfon.ico` должен лежать рядом с `resource.rc`.

## 🧩 Технические детали

- **Стек:** C, чистый WinAPI (WinHTTP, IDesktopWallpaper / IActiveDesktop, Shell_NotifyIcon, реестр, DPAPI, WIC, DWM, uxtheme).
- **Сеть:** WinHTTP со своим cookie-jar по доменам, браузероподобные заголовки, авто-ретрай на 502/503/504, переключение com↔ru при сбоях.
- **Обои:** `IDesktopWallpaper` (по монитору, режим «заполнение») с фолбэком на `IActiveDesktop` / `SystemParametersInfo`.
- **Потоки:** сетевые операции в фоне, интерлок `g_busy` не даёт операциям накладываться.
- **Фолбэк:** при недоступности сайта или исчерпании лимита обои берутся из избранного.

## 📝 Замечания

- Настройки в реестре привязаны к учётной записи Windows. При переносе на другую
  машину/учётку логин и пароль вводятся заново (следствие защиты DPAPI, сделано намеренно).
- Дневные лимиты загрузок регулирует сам сайт goodfon (больше — после авторизации).
  При исчерпании лимита обои берутся из избранного.
