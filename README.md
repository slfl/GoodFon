# GoodFon — автосмена обоев рабочего стола

**Версия 2.3.** Небольшое нативное приложение для Windows, которое автоматически меняет обои
рабочего стола, подбирая случайные изображения с сайта [goodfon.com](https://www.goodfon.com)
/ [goodfon.ru](https://www.goodfon.ru). Живёт в системном трее, ничего лишнего
не показывает, потребляет минимум ресурсов. Версия файла (2.3.0.0) и копирайт
зашиты в exe и видны в свойствах файла Windows.

Написано на чистом C с использованием только WinAPI (без сторонних runtime и
зависимостей). Размер бинарника — около 0.5 МБ.

**Автор:** Mansi / slfl@mail.ru
**Репозиторий:** https://github.com/slfl/GoodFon

---

## Возможности

- **Автосмена обоев по таймеру** — интервал 5 / 10 / 30 / 60 минут.
- **35 встроенных тем** — природа, пейзажи, город, космос, животные, авиация,
  абстракции, игры, ai-art и другие (см. полный список ниже).
- **Выбор разрешения с тирами** — HD / Full HD / 2K / 4K / «Оригинал».
  Приложение подбирает картинку в пределах выбранного тира (подробнее ниже).
- **Избранное (Favorite)** — понравившиеся обои добавляются в личную папку и
  синхронизируются с избранным на сайте. Каждая N-я смена (5 / 10 / 15 / 20)
  берётся из избранного.
- **Синхронизация избранного при старте** — при запуске приложение сверяет
  локальную папку избранного с избранным на сайте и удаляет то, что вы убрали
  с сайта (подробнее ниже). Работает тихо, видно только в режиме `-debug`.
- **Настойчивый поиск разрешения** — если для темы редко встречается нужное
  разрешение, приложение перебирает до 40 картинок, прежде чем взять локальную.
- **Хранение настроек в реестре** `HKCU\Software\GoodFon`.
- **Безопасное хранение пароля** — шифрование Windows DPAPI (не открытый текст,
  не украсть с другой машины/учётки).
- **Автозапуск с Windows** — через ветку автозапуска текущего пользователя.
- **Кастомная иконка** в трее и окнах.
- **Тихий релизный режим** — по умолчанию логи не пишутся; включаются флагом `-debug`.

## Установка и запуск

1. Скопируйте `GoodFon-x64.exe` (или `GoodFon-x86.exe` для 32-битной системы)
   в любую папку, например `C:\Program Files\GoodFon\` или переносную папку.
2. Запустите двойным кликом — появится иконка в системном трее.
3. Правый клик по иконке → **«Изменить логин и пароль…»** — введите данные
   учётной записи goodfon (нужны для доступа к разделам с авторизацией и для
   снятия дневного лимита загрузок).
4. Настройте тему, разрешение и интервал через контекстное меню.

При первом запуске, если в реестре ещё нет настроек, создаются значения по
умолчанию (тема nature, Full HD, интервал 10 минут).

## Меню в трее (правый клик по иконке)

| Пункт | Действие |
|---|---|
| Сменить обои сейчас | Немедленная смена обоев |
| Добавить в избранное ♥ | Текущие обои → в папку Favorite и в избранное на сайте |
| Убрать из избранного ♡ | Убрать текущие обои из избранного |
| Пауза | Приостановить автосмену |
| Настройки | Открыть окно настроек |
| Выход | Закрыть приложение |

Трей содержит только быстрые действия. Всё остальное — тема, интервал,
разрешение, «из избранного», авторизация, язык, оформление, автозапуск,
уведомления, обновления — перенесено в окно **«Настройки»**.

Пункт **«Настройки»** открывает отдельное окно с навигацией слева и
содержимым справа (одно окно, правая часть сменяется). Изменения применяются
сразу; внизу — кнопка «Закрыть».

| Раздел (слева) | Содержимое (справа) |
|---|---|
| Картинки | Тема, интервал смены (5 мин…24 ч), разрешение, «из избранного» (без избранного / каждая N-я), сколько временных картинок хранить, папка картинок (рядом с приложением / своя) |
| Аккаунт | Логин, пароль, кнопки «Войти» / «Выйти», ссылка «Регистрация на сайте». После входа под ссылкой появляется статус: зелёный — успех, красный — ошибка (пропадает при закрытии окна). Ниже — блок профиля с сайта: аватарка (32×32), имя, рейтинг и счётчики (обои / скачивания / комментарии). Без входа — нули и иконка приложения |
| Дополнительно | Уведомления, автозапуск, домен сайта (авто/.ru/.com), язык (RU/EN), оформление (светлая/тёмная) |
| Обновления | Текущая версия, автопроверка (выкл/час/день/неделя), проверка при запуске, автоустановка, уведомления о новых версиях, кнопка «Проверить обновления» |
| О приложении | Краткое описание, версия, копирайт автора |

Слева у разделов — иконки (Segoe MDL2 Assets). Кнопки окна отрисованы вручную
(owner-draw): скруглённые углы, плоский стиль, акцентная основная кнопка, цвета
подстраиваются под светлую/тёмную тему. Выпадающие списки в тёмной теме
перекрашиваются целиком (включая кнопку-стрелку). Меню в трее в тёмной теме тоже
рисуется тёмным (owner-draw), в светлой — обычным системным.

Тёмная/светлая тема оформления выполнена в стиле Windows 10/11: тёмный
заголовок через DWM, тёмный режим контролов через `uxtheme`, цвета фона и текста
через `WM_CTLCOLOR*`. Выбор сохраняется в реестре (`AppTheme` = `light` / `dark`)
и применяется при запуске — и к окну настроек, и к меню трея.

**Статистика профиля.** На странице «Аккаунт» после входа подгружается блок с сайта:
аватарка (32×32), имя, рейтинг и счётчики (обои / скачивания / комментарии). Данные
кэшируются в реестре (`profile_stats`) и показываются мгновенно при открытии окна, а
в фоне перечитываются с сайта и обновляются, если изменились. Если сайт недоступен —
берётся кэш; если вход не выполнен — показываются нули и иконка приложения. Аватарка в
формате WebP декодируется через WIC (нужен встроенный кодек Windows 10/11); если декод
не удался, показывается иконка приложения.

Двойной клик по иконке в трее — тоже смена обоев.

**Несколько мониторов.** Обои ставятся через `IDesktopWallpaper` в режиме
«заполнение», и в расширенном рабочем столе на каждый монитор кладётся своя
картинка (берутся соседние из папки), а не одна на все экраны. При выборе
разрешения панорамные/ультраширокие варианты (напр. 3840×1080) отсекаются по
соотношению сторон — берётся картинка, близкая к формату монитора.

## Аргументы командной строки

Порядок и наличие дефисов не важны, флаги комбинируются.

| Запуск | Что делает |
|---|---|
| `GoodFon.exe` | Трей-режим, **без логов** (релизный режим) |
| `GoodFon.exe -debug` | Трей-режим + логирование в `goodfon.log` (или в консоль при запуске из терминала) |
| `GoodFon.exe update` | Разовая смена обоев и выход |
| `GoodFon.exe favorite` | Добавить текущие обои в избранное и выйти |
| `GoodFon.exe unfavorite` | Убрать текущие обои из избранного и выйти |
| `GoodFon.exe update -debug` | Разовая операция с логированием (для отладки) |

Без флага `-debug` файл лога не создаётся вообще — экономия места и отсутствие
лишних операций записи. С флагом лог пишется рядом с exe, с ограничением 1 МБ
(при превышении файл обнуляется).

## Разрешение: как работают тиры

Для выбранного разрешения приложение принимает картинки в диапазоне
«от выбранного тира до следующего» (не включая следующий) и берёт из них
наибольшее — то есть лучшее качество в пределах тира.

| Выбрано | Подходящий диапазон разрешений |
|---|---|
| HD `1280x720` | `[1280x720 … 1920x1080)` |
| Full HD `1920x1080` | `[1920x1080 … 2560x1440)` |
| 2K `2560x1440` | `[2560x1440 … 3840x2160)` |
| 4K `3840x2160` | `[3840x2160 … 7680x4320)` |
| 8K `7680x4320` | `[7680x4320 … 10240x5760)` |
| Оригинал | максимальное доступное (без ограничений) |

То есть при выборе 4K подойдут, например, `4000x2667`, `4368x2730`, `5184x3456`,
но не «настоящее» 8K. Если в теме нет ни одной картинки нужного тира — она
пропускается, и поиск продолжается дальше.

### О безопасности пароля

Пароль хранится **зашифрованным через Windows DPAPI** (`CryptProtectData` /
`CryptUnprotectData`), а не в открытом виде и не в виде хэша.

Почему именно шифрование, а не SHA-256: сайту goodfon при входе нужно отправить
**настоящий пароль** (сервер сам хэширует его у себя и сверяет). Из
необратимого хэша исходный пароль восстановить нельзя, поэтому логиниться с
хэшем невозможно. DPAPI решает обе задачи сразу: пароль лежит в защищённом
виде (привязан к учётной записи Windows, не читается на другой машине/под
другим пользователем), но приложение под вашей учёткой расшифровывает его для
входа. Это правильный инструмент для клиента, который обязан «предъявить»
пароль удалённому серверу.

## Синхронизация избранного

При каждом запуске приложение один раз сверяет локальную папку избранного с
избранным вашего аккаунта на сайте (`/user/<login>/favorite/`) и **удаляет
локально те картинки, которых больше нет в избранном на сайте**. Это
одностороннее удаление: если вы убрали картинку из избранного на сайте (с
любого устройства), при следующем запуске она удалится и локально.

Что синхронизация **не** делает: ничего не скачивает. Если на сайте картинок
больше, чем локально — локальная папка не пополняется (докачки нет).
Сопоставление идёт по имени файла (slug), темы/разделы значения не имеют —
чистятся все подпапки `Favorite\`.

Синхронизация устроена максимально осторожно, чтобы не удалить лишнее:

- Выполняется только при успешном входе (иначе пропускается).
- Список с сайта читается со всех страниц. Сайт нередко отдаёт часть карточек
  как ошибку `502` — такие страницы дочитываются повторно и объединяются, пока
  не соберётся полный набор (24 карточки на странице).
- **При любом сомнении** (сеть недоступна, страница не собралась полностью,
  список пуст) синхронизация **отменяется целиком и ничего не удаляет** —
  лучше пропустить один раз, чем удалить то, что на самом деле в избранном.

Весь процесс тихий; подробности видны только при запуске с флагом `-debug`.

## Аккаунт

Работа с учётной записью goodfon вынесена в подменю **«Аккаунт»**, состав
которого зависит от того, выполнен ли вход:

- **Не выполнен вход:** «Регистрация на сайте…» и «Авторизация…».
- **Выполнен вход:** «Авторизация…» и «Выйти из аккаунта».

«Регистрация на сайте…» открывает страницу регистрации goodfon в браузере.
Регистрация прямо в приложении не поддерживается: на форме регистрации стоит
Google reCAPTCHA, которую по назначению нельзя пройти автоматически. После
регистрации на сайте войдите в приложении через «Авторизация…», указав тот же
логин и пароль.

«Авторизация…» открывает окно ввода логина и пароля (пароль шифруется DPAPI,
кэш сессий сбрасывается). «Выйти из аккаунта» — локальный выход: приложение
очищает логин, пароль и кэш сессий в реестре. На сайте при этом ничего не
происходит (сессия в браузере не затрагивается).

## Язык интерфейса

Приложение полностью локализовано на **русский** и **английский** — переводятся
и элементы меню/диалогов, и сообщения в логах. Язык переключается в подменю
**«Язык / Language»** и сохраняется в реестре в значении `Language`
(`russian` / `english`). Смена применяется сразу: меню, подсказка в трее и
последующие записи в лог выводятся на выбранном языке.

## Обновление с GitHub

Пункт меню **«Проверить обновления»** сравнивает **хэш** текущего exe с exe,
лежащим в репозитории:

```
https://github.com/slfl/GoodFon/raw/refs/heads/main/GoodFon-x64.exe
```

Приложение скачивает удалённый файл во временный, считает его хэш (FNV-1a 64)
и хэш текущего exe. Если они **совпадают** — обновляться нечего, временный файл
удаляется. Если **различаются** — проверяет, что скачан настоящий exe
(сигнатура `MZ` и разумный размер), и ставит его **с заменой работающего файла**:

1. переименовывает себя в `<exe>.old` (работающий файл переименовать можно);
2. кладёт скачанный файл на место исходного;
3. запускает новую версию и завершает текущую.

На следующем старте новый экземпляр удаляет оставшийся `<exe>.old`.

Отдельный файл версий не нужен: после установки локальный exe становится
байт-в-байт как в репозитории, поэтому хэши совпадают и повторных обновлений не
происходит. Чтобы выпустить обновление, достаточно залить новые
`GoodFon-x64.exe` / `GoodFon-x86.exe` в корень репозитория https://github.com/slfl/GoodFon

## Автозапуск с Windows

Пункт меню «Автозапуск с Windows» записывает/удаляет значение в стандартной
ветке автозапуска текущего пользователя (права администратора не требуются):

```
HKCU\Software\Microsoft\Windows\CurrentVersion\Run
    GoodFon = "<полный путь>\GoodFon.exe"
```

При входе в систему Windows запускает exe, он читает настройки из реестра и
уходит в трей. Планировщик заданий не используется.

## Сборка из исходников

Исходники лежат в папке `src/`. Код совместим с компиляцией и как **C**, и как
**C++** (MSVC компилирует `.c` как C, что также поддерживается).

### Вариант 1. Visual Studio + CMake (рекомендуется)

```bat
cd src
cmake -B build -A x64
cmake --build build --config Release
```

Для 32-битной сборки: `cmake -B build32 -A Win32`.

CMake сам подключит ресурс-компилятор для `resource.rc`. Файл `goodfon.ico`
должен лежать рядом с `resource.rc`.

### Вариант 2. Visual Studio вручную

1. Создайте проект Win32 GUI Application.
2. Добавьте `main.c` и `resource.rc` в проект.
3. Убедитесь, что `goodfon.ico` лежит рядом с `resource.rc`.
4. В свойствах включите `/utf-8` (в исходнике есть русские строковые литералы).
5. Линкер подхватит нужные библиотеки автоматически через `#pragma`/CMake;
   при ручной настройке добавьте: `winhttp shlwapi ole32 shell32 user32
   advapi32 gdi32 crypt32`.

### Вариант 3. MinGW-w64 (как собирались эти бинарники)

x64:
```sh
x86_64-w64-mingw32-windres resource.rc -O coff -o resource.o
x86_64-w64-mingw32-gcc -municode -mwindows -O2 -Wall -o GoodFon-x64.exe \
    main.c resource.o \
    -lwinhttp -lshlwapi -lole32 -lshell32 -luser32 -ladvapi32 -lgdi32 -lcrypt32
```

x86:
```sh
i686-w64-mingw32-windres resource.rc -O coff -o resource.o
i686-w64-mingw32-gcc -municode -mwindows -O2 -Wall -o GoodFon-x86.exe \
    main.c resource.o \
    -lwinhttp -lshlwapi -lole32 -lshell32 -luser32 -ladvapi32 -lgdi32 -lcrypt32
```

    ├── resource.h         заголовок ресурсов (ID иконки)
    ├── goodfon.ico        иконка приложения (16…128 px)
    └── CMakeLists.txt      сборка через CMake

## Технические детали

- **Язык / стек:** C, чистый WinAPI (WinHTTP, IActiveDesktop, Shell_NotifyIcon,
  реестр, DPAPI). Без CRT-зависимостей сверх стандартных.
- **Сеть:** WinHTTP с собственным cookie-jar по доменам, браузероподобные
  заголовки, авто-ретрай на 502/503/504, переключение com↔ru при сбоях.
- **Обои:** установка через `IActiveDesktop` с проверкой и фолбэком на
  `SystemParametersInfo`.
- **Разбор HTML:** сайт отдаёт минифицированный HTML с неэкранированными
  атрибутами — парсер это учитывает.
- **Потоки:** сетевые операции в фоновом потоке, интерлок `g_busy` не даёт
  операциям накладываться; `rand()` сидируется отдельно в каждом потоке.
- **Локальный фолбэк:** при недоступности сайта или исчерпании лимита обои
  берутся из папки избранного.

## Замечания

- Настройки хранятся в реестре и привязаны к учётной записи Windows. При
  переносе приложения на другую машину/учётку логин и пароль нужно ввести
  заново (это следствие защиты DPAPI и сделано намеренно).
- Дневные лимиты загрузок регулируются самим сайтом goodfon (больше — после
  авторизации). При исчерпании лимита приложение берёт картинки из избранного.
